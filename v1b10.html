What the interface is showing you

The interface is not trying to be friendly.

It is trying to be truthful.

Most software hides complexity behind a single surface. One window. One canvas. One place where things happen. Anything else is tucked away so you can feel oriented. That choice is comforting, but it comes with a cost: you only see the part of the system someone decided you should see.

This interface refuses that arrangement.

Multiple panes exist because multiple things are happening at once. Network view shows relationships. Parameter panes show conditions. Viewers show symptoms. Timelines show persistence. None of these replace the others, because none of them tell the whole story on their own.

Viewers cause the most confusion. They look authoritative. Bright. Immediate. Convincing. What they display is not truth; it is output at a particular moment under particular conditions. Treating them as ground truth leads to bad decisions fast.

Parameters feel overwhelming for the opposite reason. They expose everything. No opinionated defaults. No gentle slope from beginner to expert. Every lever that exists is visible, because the system does not know which ones matter to you.

The network editor is not a workspace in the usual sense. It is a map of obligation. Each wire represents something that must be honored continuously. Each container represents a role that never stops performing.

This is why rearranging panes does not simplify the system. It only changes which part you are listening to more closely. Complexity does not disappear when you hide it. It continues operating off-screen.

People often ask where to look when something feels wrong. The answer depends on what kind of wrong it is. Structural problems show up in the network. Behavioral problems show up in parameters. Perceptual problems show up in viewers. Time-related problems show up everywhere at once.

The interface does not prioritize any of these perspectives because the system itself does not. It expects you to move between them, correlating behavior rather than trusting a single surface.

Once this is accepted, the interface stops feeling busy and starts feeling honest. It is not presenting a simplified story. It is presenting multiple partial truths and letting you decide which one matters at the moment.

Nothing here is ornamental. Every pane exists because the system needs a place to reveal something it cannot compress into one view.
