How text and tables quietly run everything

So far, everything has moved because numbers pushed it.

Numbers feel powerful, but they are forgetful. They exist for a moment, do their job, then vanish into the next frame.

Something else is happening underneath.

Decisions are being stored. Names are being remembered. Relationships are being held in place. That work is not done by images or sound.

It is done by text and tables.

Move your mouse to an empty area to the left of your existing network.

Press Tab.

Type one of the following and place it there:

  • Table
  • Text
  • Constant (DAT)

Any of these creates stored information. Pick one.

Open the viewer.

This node is not reacting to time. It is waiting. Nothing changes unless something tells it to.

That waiting is the point.

If you chose a Table, add a few values by clicking into the cells.

If you chose Text, type a few words or numbers.

If you chose Constant, add a few named fields.

You just created memory.

This data does not care about frames, audio, or visuals. It exists outside the flow and can be consulted at any time.

Press Tab.

Add one of the following to the right:

  • DAT to CHOP
  • DAT to SOP
  • DAT to TOP

This is another border crossing. Pick any.

Wire the text or table into the new node.

Stored information has entered the moving system.

Take the output and wire it into something you already built:

  • a Transform parameter
  • a Level or Color parameter
  • a Switch index

The system now responds to decisions instead of impulses.

Change a value in the table or text. Everything downstream updates immediately.

Nothing animated the change. Nothing eased it. The update happened because memory changed.

This is how logic enters the system.

Images persuade the eye. Sound persuades the body. Tables persuade the structure.

Most large networks are not driven by noise or audio. They are driven by quiet collections of values that decide what is allowed to happen.

When something feels impossible to coordinate, the missing piece is often not a clever node, but a place to write something down and reuse it.
