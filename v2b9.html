How feedback turns a setup into a system

So far, everything has moved forward.

Sources push. Transforms respond. Images appear. Numbers describe them. The flow has direction. That direction has been polite.

A system begins when something is allowed to look back.

Feedback is not repetition. It is influence traveling upstream.

Start from a setup where an image is already affecting something else.

You should have a number that came from an image, sound, or geometry.

Right now that number probably feeds forward into another behavior. That is safe. Feedback is not.

Find a parameter earlier in the network that could accept a number:

  • Noise amplitude
  • Transform scale or rotation
  • Level brightness

Pick something upstream.

You are about to create a loop.

Wire the number back into that earlier parameter.

One of two things happens.

Either the system becomes unstable immediately, or it settles into a new behavior. Both outcomes are correct.

This is the first time the system reacts to itself.

If it explodes, that is not failure. It means amplification exists. Something downstream is shouting back louder than the source expects.

Insert one of the following anywhere in the feedback path:

  • Lag
  • Filter
  • Math (reduce the range)

The system calms down.

What you are seeing is not a trick. It is balance being negotiated in real time.

Feedback reveals things that forward motion hides: sensitivity, inertia, thresholds, memory.

Even without storing anything, the loop remembers. Its current state depends on what just happened.

This is why feedback is never introduced early. It requires respect for scale, timing, and pressure.

Once it exists, the network is no longer a chain. It is a conversation.

Many behaviors people call “alive” come from nothing more than a system being allowed to hear itself.

If something starts to feel responsive instead of reactive, you are probably looking at a loop that has been given room to breathe.
