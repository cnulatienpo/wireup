How control enters the system

Everything so far has decided things for itself.

Numbers moved because time moved. Images changed because values changed. Even feedback reacted according to rules already in place.

Control begins when something outside the system is allowed to speak.

Start from a network where something visible is already happening.

Leave it running.

You are going to introduce intention.

Move your mouse to an empty area below the network.

Press Tab.

Type one of the following and place it there:

  • Slider (CHOP)
  • Button (CHOP)
  • Keyboard In
  • Mouse In

Pick something you can physically manipulate.

Open the viewer.

This node does nothing by itself. It waits for you.

That waiting is different from everything else you have added so far.

Interact with the control.

Drag the slider, click the button, move the mouse, press a key.

A value changes only when you act.

This is not time. This is choice.

Take the output of the control node.

Wire it into something that already exists:

  • a Switch index
  • a Transform parameter
  • a Level brightness
  • a Noise amplitude

The system now waits for instruction.

When you move, it responds. When you stop, it freezes in its last decision.

This is the first time the system defers.

Control nodes do not generate content. They authorize it.

Most interactive systems are not reactive because of complexity. They are reactive because someone, somewhere, is allowed to say “now.”

From here on, any time you want meaning to change without randomness, control is the door it enters through.
