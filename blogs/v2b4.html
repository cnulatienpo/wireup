How space becomes images

This guide explains GPU space vs data space in TouchDesigner in plain language. It focuses on practical decisions you can make while building and debugging TouchDesigner networks.

TouchDesigner operates in two primary territories: GPU space and CPU data space.

GPU space handles images, textures, geometry, and rendering. CPU data space handles tables, scripts, and structured values.

TOPs and most 3D rendering operations occur on the GPU. DATs and many scripting operations occur on the CPU.

These spaces behave differently.

GPU processing is massively parallel and optimized for pixels and vertices. CPU processing is sequential and optimized for logic and structured data.

When you move data from one space to another, translation happens. That translation costs time.

For example, converting a TOP to CHOP values or exporting a texture into a DAT table forces GPU data into CPU memory.

Frequent crossing between GPU and CPU can reduce performance significantly.

The question is not which space is better. The question is which space matches the task.

Image processing, feedback loops, compositing, and rendering belong on the GPU.

String manipulation, structured logic, file operations, and table lookups belong on the CPU.

If something feels unexpectedly slow, examine where it is being processed.

Efficiency often comes from minimizing unnecessary border crossings.

Understanding this separation allows you to design networks that remain responsive even as complexity increases.

FAQ

What is GPU space in TouchDesigner?

GPU space refers to operations performed on the graphics card, including image processing, texture manipulation, and 3D rendering.

Why is TOP processing different from DAT processing?

TOP processing happens on the GPU and is optimized for pixel data, while DAT processing happens on the CPU and is optimized for structured text and logic operations.

Should I process this on GPU or CPU?

If the task involves images, textures, or rendering, use GPU operators. If it involves structured data, scripting, or file logic, use CPU-based operators.
