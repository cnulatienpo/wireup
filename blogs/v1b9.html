The network is the program

This guide explains data vs image vs geometry in TouchDesigner in plain language. It focuses on practical decisions you can make while building and debugging TouchDesigner networks.

There is no hidden script behind the scenes.

What you see wired together is not a representation of the program. It is the program.

Structure and behavior live in the same place.

Each connection is a relationship that must be honored continuously. Remove it and behavior changes immediately.

Behavior emerges from arrangement, not from a single important node.

Small structural changes can have larger effects than dramatic parameter tweaks.

Refactoring feels physical because responsibilities shift while everything keeps running.

The network is not documentation describing something else. It exists to be evaluated.

Problems usually live in how influence travels across the structure, not inside one parameter.

Effort shifts from searching for the right setting to arranging flow correctly.

The program is spelled out in wires, dependencies, and repetition across frames.

FAQ

What is the difference between data and image in TouchDesigner?

Data refers to numeric or structured information (CHOPs and DATs). Images are pixel-based data processed in TOPs.

Can I turn CHOP channels into geometry?

Yes. CHOP data can drive SOP parameters or be converted into geometry using conversion operators.

How do I convert a TOP to CHOP values?

You can use a TOP to CHOP operator to extract pixel values into channels for further processing.
